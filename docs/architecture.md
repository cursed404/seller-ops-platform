# Архитектура

## В двух словах

Проект поделен на две части:

- API принимает запросы
- worker в фоне обрабатывает процесс

Между ними есть Redpanda. Состояние и история лежат в Postgres. Redis используется для идемпотентности у внешних действий. Метрики отдаются в Prometheus, а смотреть их удобно через Grafana.

## Из чего состоит система

### API

Что делает:

- принимает инциденты
- создает `incident` и процесс обработки
- публикует стартовые события
- отдает текущее состояние процесса
- принимает решение по подтверждению

### Worker

Что делает:

- слушает события из Redpanda
- запускает LangGraph-процесс
- пишет шаги, события, tool invocations и retries в базу
- завершает процесс или ставит его на паузу

### PostgreSQL

Тут лежит все важное:

- сами инциденты
- запуск процесса
- шаги процесса
- события
- approvals
- retries
- регламенты
- тестовые бизнес-данные по заказам, остаткам, ценам и задачам синхронизации

### Redis

Нужен, чтобы не отправить одно и то же внешнее действие несколько раз, например запрос на сверку или изменение цены.

### Redpanda

Нужен, чтобы API не тянул на себе выполнение процесса. API просто принимает запрос и публикует событие, а дальше уже worker делает остальную работу.

## Как течет запрос

1. Приходит `POST /api/v1/incidents`
2. API сохраняет инцидент и процесс
3. Публикуется `incident.received`
4. Worker забирает событие и запускает процесс
5. Процесс идет по шагам: классификация, сбор контекста, выбор регламента, оценка риска, действие, проверка результата
6. Если нужно подтверждение, процесс уходит в `waiting_for_approval`
7. После approval worker продолжает выполнение

## Какие события публикуются

- `incident.received`
- `workflow.created`
- `workflow.started`
- `incident.classified`
- `context.collected`
- `runbook.selected`
- `action.requested`
- `approval.required`
- `approval.received`
- `action.executed`
- `workflow.completed`
- `workflow.failed`

Формат у событий один и тот же:

```json
{
  "event_id": "uuid",
  "event_type": "incident.classified",
  "occurred_at": "2025-10-14T03:30:00Z",
  "correlation_id": "uuid",
  "trace_id": "uuid",
  "workflow_id": "uuid",
  "incident_id": "uuid",
  "payload": {}
}
```

## Что замокано, а что нет

Замокано:

- внешние marketplace write API
- тикетная система
- LLM по умолчанию

Работает:

- API
- worker
- очереди
- LangGraph-процесс
- approvals
- retries
- Postgres persistence
- метрики и логирование
